Developer Productivity Dashboard

An interactive data visualization dashboard that aggregates and displays developer metrics from GitHub, helping engineering teams identify bottlenecks and improve their development workflow.

Kibana Elasticsearch Python PostgreSQL Apache Airflow Docker GitHub API
The Challenge

What Problem Did I Solve?

Prunedge Development Technologies had a growing engineering team spread across multiple projects and repositories. Management needed visibility into developer activity, but manually tracking commits, pull requests, and code reviews across dozens of GitHub repositories was time-consuming and error-prone.

The Core Problem

Engineering managers were spending 3-4 hours weekly compiling developer activity reports manually. There was no centralized view of team productivity, making it difficult to identify who needed support, which projects were bottlenecked, and where process improvements could be made.

The team needed a solution that would automatically collect data from GitHub, process it into meaningful metrics, and present it in an easy-to-understand visual format that could be accessed by both technical and non-technical stakeholders.

The Solution

What I Built

I designed and built an end-to-end data pipeline that automatically pulls developer activity from GitHub, transforms it into actionable metrics, and displays everything in a real-time Kibana dashboard.

Impact

Results & Outcomes

Key Achievements

The dashboard reduced manual reporting time from 3-4 hours per week to nearly zero. Engineering managers could now access real-time metrics at any time, and the visualization helped identify a bottleneck in the code review process that was slowing down deployments by an average of 2 days.

Beyond the immediate productivity gains, the project established a data infrastructure that could be extended to track other metrics. The same pipeline architecture was later adapted to monitor application performance and customer support tickets.

Reflection

What I Learned

This project was my first experience building a production data pipeline from scratch. Beyond the technical skills, I learned valuable lessons about working with stakeholders and delivering data products that actually get used.

  • Start with the end user: I spent time with engineering managers understanding what decisions they needed to make before writing any code. This ensured the dashboard showed metrics that actually mattered.
  • Data quality is everything: Raw GitHub data needed significant cleaning. I learned to build validation checks into the pipeline to catch issues before they reached the dashboard.
  • Airflow is powerful but complex: Managing DAG dependencies and handling failures gracefully took more time than expected. I now appreciate the importance of proper error handling in ETL workflows.
  • Docker simplifies deployment: Containerizing the ELK stack (Elasticsearch, Logstash, Kibana) made it easy to replicate the environment and onboard other developers to the project.
  • Documentation matters: I maintained a wiki documenting the data schema, pipeline architecture, and common troubleshooting steps. This proved invaluable when handing off the project.
